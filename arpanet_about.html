About the Arpanet Reconstruction Project

Overview

This project builds on three decades of computer archeaology done by dozens (probably, hundreds) of people. There are projects to write accurate simulators for vintage computers, like the PDP-10. There were projects to recover their old operating systems and software. There were a few projects, building on each other, to write a IMP simulator and make the old firmware run on it. Then, there was a project to create an Arpanet driver (NCP) for Linux. Projects exist to make hardware replicas of all this vintage hardware.

So, now, all that was left to do is to integrate it all. Let simulated mainframes connect over simulated copper wires to simulated IMP routers. And hook up the IMPs with simulated leased-line connections. That is what this project is about. But 99.9% of the work was done in all these projects above. It is silly to make a Hall of Fame, because we'd be sure to leave out some crucial Famers. That having been said, to the right is the Hall of Fame of making the 1972 Arpanet come alive again.

Is this really the Arpanet circa 1972/1973?

Yes. You are telnetting through simulated IMP routers with their actual firmware from 1973. The IMP simulator is so accurate that we needed the source code to figure out all the little tricks inside. And yes, if you log in to the MIT AI Lab's PDP-10, you are running the actual ITS operating system. When recovered from old crumbling archive magtapes, it had Arpanet code in it. And when the simulator implemented the Arpanet hardware, it connected. The programs on these servers are the originals, gathered from university archives and people's storage boxes.

No. These are hardware replicas and software simulations, tricked into believing that TCP/IP ports are the copper wires connecting them. It's not real kilowatt-munching old Big Iron (fun fact: just running one disk pack of a PDP-10 would cost you 50 cents an hour in electricity). But would you have teleported from 1972 to this reconstructed Arpanet, you would feel right at home. As if there had been no changes to software, operating systems and Arpanet node list in 50 years.

Is this the entire Arpanet?

No. We are recovering node by node. Only a few nodes are up. But as luck would have it, they happen to be the nodes that attracted by far the most traffic. And are by far the most important (and fun). Many nodes were User nodes, not accessible from the outside. Others were accessible, but locked behind passwords.

And the nodes that are up now, the MIT and Stanford systems, are not only the most important, their reconstruction is also very real. As in, they run the unmodified operating systems, with recovered original software. But of course, much has been lost. Sometimes forever, sometimes because old storage media have yet to be recovered or even discovered. Time will tell how far this goes. People are working on UCLA's first node on the Arpanet. The actual operating system they ran is probably lost. Maybe because it was called SEX? Who knows. But for such systems, there is work being done to come at least close. Run another nearly period-correct operating system instead, rewriting some of the old programs that used to run on SEX. And for some systems, the highly specific Arpanet drivers (NCPs) have been lost. Then, we can interface their consoles through a newly developed Linux NCP driver. The user experience is no different then. But it will not all be ancient bits.

So as the reconstructed Arpanet increases by adding more and more nodes, its fidelity will decrease a little bit, and that is OK in our opinion. As long as it will be properly disclaimed. But rest assured: telnet from the 1972 Hilton ICCC conference into MIT or Stanford, you're surfing on nothing but recovered ancient bits.

Computer Archeology

Over the past decade, Retrocomputing and Vintage Computer hobbyists have dug farther back in time beyond their the good old homecomputers from the 1980s. That is how Computer Archaeology came to be a related, yet different hobby. Preserving the hands-on experience of 1960s computer systems. We gather around our Obsolescence Guaranteed web site, and the various Google Groups. The idea is to keep these old systems alive, through hardware replicas (Blinkenlights!) and a hands-on experience of how these machines felt. Because an unpowered metal box in a museum tells you nothing. And there is a lot worth preserving. But equally important: this can be great fun. Because with the amount of computer-savviness that every 21st-century computer user has, it turns out the systems of our Forefathers were not that complicated to use after all. With the benefit of hindsight, some patience and a few manuals, it must be added.

Help wanted

You will see we have dedicated web pages for each of the 1973 Arpanet nodes, describing the organisations, their computers, the people and anecdotes that bring them to life. Arpanet was a social network... but this is a huge, huge subproject: one for each node. The current content is just placeholder text to get things started. But if you have information, photos, or even: if you or your organisation want to adopt a node page: contact us! Either by joining the Google Group, or by dropping us an email. You will find contact information on www.obsolescence.dev/about . This project can really use help from developers, people with experience from back in the day - and from digital archaeology hobbyists who want to delve into each node's history. We're all hobbyists in this project. Proudly so.
